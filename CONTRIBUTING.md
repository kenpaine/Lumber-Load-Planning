# Contributing

Thank you for your interest in this project!

## How to contribute

This repository is maintained by a single owner. **All changes require prior approval** — no one
(other than the repository owner) may push directly to `master`.

### Steps

1. **Open an issue first** — describe what you want to change and why.  
   Wait for a response before writing any code; the change may already be planned, out of scope, or handled differently.

2. **Fork the repository** and create a feature branch off `master` in your fork.

3. **Make your changes** on that branch. Keep changes focused — one feature or fix per pull request.

4. **Open a pull request** from your branch into `master` here.  
   Include a clear description of what changed, why, and how you tested it.

5. **Wait for review.** The owner will review and either approve, request changes, or close the PR with an explanation.  
   PRs are not merged automatically — every merge is a deliberate decision by the owner.

## Ground rules

- Be respectful and constructive in all issue and PR comments.
- Do not push directly to `master` — the branch is protected and will reject direct pushes.
- Do not open PRs for purely cosmetic changes (whitespace, comment rewording) without prior discussion.
- Security issues must be reported privately — see [SECURITY.md](SECURITY.md).

## What gets accepted

The bar is practical utility for rail car lumber load planning:

- Bug fixes that affect solver correctness, Excel formula accuracy, or HTML app behavior
- Accuracy improvements (new pack sizes, layout edge cases)
- Genuine UX improvements agreed upon in an issue first

Refactors, style preferences, or feature additions that significantly expand scope are unlikely to be merged
without prior discussion.
